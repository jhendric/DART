PUBLIC ROUTINES

A note about documentation style. Optional arguments are enclosed in brackets [like this].

call yyyroutine1( var1, var2, var3 [,global] )

DESCRIPTION

Returns the resolution of compute domain for either the current processor or the global domain.

This is the best thing since sliced bread. All you have to do is throw some arguments in the call and the subroutine automatically slices, dices and makes julienne fries. But wait! There's more!

Variable name   IN?OUT   meaning [units look like this]
var1 IN The initial (or base) time. [time_type]
var2 INOUT The current time. [time_type]
var3 OUT The atmospheric model/physics time step. [time_type]
global optional whatever it does, it does it really well.

DISCUSSION

Var2 is used for both input and output. Upon input, this contains the furlongs per weekday. Upon exit, we now have the number of spotted gobies per square hectare.
