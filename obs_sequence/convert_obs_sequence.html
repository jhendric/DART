OVERVIEW

This program can convert an existing observation sequence file from ascii into binary or the reverse. An ascii (formatted) file is both machine-portable and human readable, but has a larger filesize, is subject to round-off errors, and takes longer to read and write. A binary (unformatted) file is fast and precise to read and write, but cannot be moved from one machine to another if one is a little-endian machine (e.g. intel CPU) and the other is a big-endian machine (e.g. IBM PowerPC, Sun Sparc, SGI MIPS, Apple PowerPC Macs). Note that the manufacturer alone is no longer sufficient to decide which endian is correct; IBM and SGI now sell clusters using intel CPUs (e.g. SGI Altix systems) and the new Apple Macs use intel CPUs.

The input file format will automatically be detected (ascii/formatted or binary/unformatted) at run time. The output file format is controlled by the namelist in file input.nml: 
&obs_sequence_nml
   write_binary_obs_sequence = .false.  /
A false setting will create an ascii/formatted output file. A true setting will create a binary/unformatted output file.
